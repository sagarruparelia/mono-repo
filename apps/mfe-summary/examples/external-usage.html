MFE Summary - External Usage

This example demonstrates how external partner portals can embed the Summary MFE using web components with OAuth2 Client Credentials authentication.

API Routing: By default, the MFE makes same-origin API calls. Use service-base-url to route API traffic through your proxy endpoint (e.g., /api/mfe-proxy) which forwards requests to the BFF.

Example 1: Basic Usage (Same-Origin API)

<!-- Include the MFE bundle --> <script src="https://cdn.example.com/mfe-summary.umd.js"></script> <!-- Use the web component (API calls go to same origin) --> <mfe-summary member-id="member-123" persona="agent" operator-id="operator-456" operator-name="Jane Smith" ></mfe-summary>

Example 2: With Proxy Endpoint

<!-- Route MFE API calls through partner's proxy --> <mfe-summary member-id="member-789" persona="case_worker" operator-id="cw-001" operator-name="John Doe" service-base-url="/api/mfe-proxy" ></mfe-summary> <!-- Partner proxy forwards: /api/mfe-proxy/* -> BFF -->

Example 3: Dynamic Attribute Updates

// JavaScript to dynamically update member ID const summaryEl = document.querySelector('mfe-summary'); summaryEl.setAttribute('member-id', 'new-member-456');

Integration Architecture

Partner Portal Partner Backend BFF +-------------+ +----------------+ +---------+ | <mfe-summary| same-origin | | OAuth2 CC | | | ... | ---------------> | /api/mfe-proxy | ------------> | /api/* | | /> | API calls | | + headers | | +-------------+ +----------------+ +---------+ Required proxy headers: Authorization: Bearer <oauth2-cc-token> X-Auth-Type: oauth2-proxy X-Partner-Id: partner-001 X-Member-Id: member-123 X-Persona: agent X-Operator-Id: operator-456 X-Operator-Name: Jane Smith

Integration Requirements

  • Authentication: Partner backend must obtain OAuth2 CC token and proxy requests to BFF
  • Proxy Endpoint: Set up a proxy route (e.g., /api/mfe-proxy) that forwards to BFF
  • Headers: Partner proxy must inject required headers (X-Auth-Type, X-Partner-Id, X-Member-Id, etc.)
  • CORS: Not needed - MFE makes same-origin calls to partner's proxy
  • Personas: Valid proxy personas: agent, config, case_worker
