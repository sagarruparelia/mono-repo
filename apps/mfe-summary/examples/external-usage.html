MFE Summary - External Usage

This example demonstrates how external partner portals can embed the Summary MFE using web components with OAuth2 Client Credentials authentication.

Example 1: Basic Usage (Agent Persona)

<!-- Include the MFE bundle --> <script src="https://cdn.example.com/mfe-summary.umd.js"></script> <!-- Use the web component --> <mfe-summary member-id="member-123" persona="agent" operator-id="operator-456" operator-name="Jane Smith" api-base="https://bff.example.com" ></mfe-summary>

Example 2: Case Worker Persona

<mfe-summary member-id="member-789" persona="case_worker" operator-id="cw-001" operator-name="John Doe" api-base="https://bff.example.com" ></mfe-summary>

Example 3: Dynamic Attribute Updates

// JavaScript to dynamically update member ID const summaryEl = document.querySelector('mfe-summary'); summaryEl.setAttribute('member-id', 'new-member-456');

Integration Requirements

  • Authentication: Partner backend must obtain OAuth2 CC token and proxy requests to BFF
  • Headers: Partner must include required headers (X-Auth-Type, X-Partner-Id, X-Member-Id, etc.)
  • CORS: Partner domain must be allowlisted in BFF configuration
  • Personas: Valid proxy personas are: agent, config, case_worker
