MFE Profile - External Usage

This example demonstrates how external partner portals can embed the Profile MFE using web components. The Profile MFE supports viewing and editing member profiles.

API Routing: By default, the MFE makes same-origin API calls. Use service-base-url to route API traffic through your proxy endpoint (e.g., /api/mfe-proxy) which forwards requests to the BFF.
Security Note: Profile editing requires step-up authentication. The BFF will require re-authentication for PUT/DELETE operations if the last auth was more than 5 minutes ago (configurable via zero-trust.step-up-auth settings).

Example 1: Agent Viewing Member Profile

<!-- Include the MFE bundle --> <script src="https://cdn.example.com/mfe-profile.umd.js"></script> <!-- Use the web component (API calls go to same origin) --> <mfe-profile member-id="member-123" persona="agent" operator-id="agent-456" operator-name="Jane Smith" ></mfe-profile>

Example 2: With Proxy Endpoint (Config Persona)

<!-- Route MFE API calls through partner's proxy --> <mfe-profile member-id="member-456" persona="config" operator-id="admin-001" operator-name="Admin User" service-base-url="/api/mfe-proxy" ></mfe-profile> <!-- Partner proxy forwards: /api/mfe-proxy/* -> BFF -->

API Contract

The Profile MFE makes the following API calls through the BFF:

// GET /api/mfe/profile/{memberId} // Response: { "memberId": "member-123", "firstName": "John", "lastName": "Doe", "email": "john.doe@example.com", "phone": "+1-555-0123", "address": { "street": "123 Main St", "city": "Anytown", "state": "CA", "zip": "90210" } } // PUT /api/mfe/profile/{memberId} // Request body: { "phone": "+1-555-9999", "address": { ... } }

Integration Requirements

  • Authentication: Partner backend must obtain OAuth2 CC token and proxy requests to BFF
  • Proxy Endpoint: Set up a proxy route (e.g., /api/mfe-proxy) that forwards to BFF
  • Headers: Partner proxy must inject required headers (X-Auth-Type, X-Partner-Id, X-Member-Id, etc.)
  • CORS: Not needed - MFE makes same-origin calls to partner's proxy
  • Personas: Valid proxy personas: agent, config, case_worker
