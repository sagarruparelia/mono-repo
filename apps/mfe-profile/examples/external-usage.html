MFE Profile - External Usage

This example demonstrates how external partner portals can embed the Profile MFE using web components. The Profile MFE supports viewing and editing member profiles.

Security Note: Profile editing requires step-up authentication. The BFF will require re-authentication for PUT/DELETE operations if the last auth was more than 5 minutes ago (configurable via zero-trust.step-up-auth settings).

Example 1: Agent Viewing Member Profile

<!-- Include the MFE bundle --> <script src="https://cdn.example.com/mfe-profile.umd.js"></script> <!-- Use the web component --> <mfe-profile member-id="member-123" persona="agent" operator-id="agent-456" operator-name="Jane Smith" api-base="https://bff.example.com" ></mfe-profile>

Example 2: Config Persona (Admin View)

<mfe-profile member-id="member-456" persona="config" operator-id="admin-001" operator-name="Admin User" api-base="https://bff.example.com" ></mfe-profile>

API Contract

The Profile MFE makes the following API calls through the BFF:

// GET /api/mfe/profile/{memberId} // Response: { "memberId": "member-123", "firstName": "John", "lastName": "Doe", "email": "john.doe@example.com", "phone": "+1-555-0123", "address": { "street": "123 Main St", "city": "Anytown", "state": "CA", "zip": "90210" } } // PUT /api/mfe/profile/{memberId} // Request body: { "phone": "+1-555-9999", "address": { ... } }

Required Proxy Headers

// Headers that partner backend must send to BFF Authorization: Bearer <oauth2-cc-token> X-Auth-Type: oauth2-proxy X-Partner-Id: partner-001 X-Member-Id: member-123 X-Persona: agent X-Operator-Id: operator-456 X-Operator-Name: Jane Smith X-Correlation-Id: <uuid>
